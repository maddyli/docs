Medplum implements the draft [FHIR GraphQL Mutation spec](https://hl7.org/fhir/R4/graphql.html#mutations). For the inputs, you would append the action (Create, Update, or Delete) to the resource type.

Here are examples of mutations for the `Patient` resource. You can test these mutations at [graphiql.medplum.com](https://graphiql.medplum.com/)

## Create Mutation[​](#create-mutation "Direct link to Create Mutation")

You can create a resource using the `[resourceType]Create` mutation.

To create a [`Patient`](/docs/api/fhir/resources/patient):

<Tabs>
  <Tab title="GraphQL">
    ```
    mutation {  # Define the fields for the resource being created  PatientCreate(    res: {      resourceType: "Patient"      gender: "male"      name: [          {              given: "Homer"          }        ]       }      )      # Specify which of the newly created fields to return in the response      {        id        gender        name {          given        }      }  }
    ```
  </Tab>

  <Tab title="Typescript">
    ```
    const patient = await medplum.graphql(`  mutation {    # Define the fields for the resource being created    PatientCreate(      res: {        resourceType: "Patient"        gender: "male"        name: [            {                given: "Homer"            }          ]         }        )        # Specify which of the newly created fields to return in the response        {          id          gender          name {            given          }        }    }`);
    ```
  </Tab>
</Tabs>

<Accordion title="Example Response">
  ```
  {  data: {    PatientCreate: {      id: 'example-id',      name: {        given: 'Homer',      },      gender: 'male',    },  },};
  ```
</Accordion>

### Aliasing the output[​](#aliasing-the-output "Direct link to Aliasing the output")

Just as with GraphQL queries, you can alias the newly created resource.

<Tabs>
  <Tab title="GraphQL">
    ```
    mutation {  # Define the fields for the resource being created, and alias as "newPatient"  newPatient: PatientCreate(    res: {      resourceType: "Patient"      gender: "male"      name: [          {              given: "Homer"          }        ]       }      )      # Specify which of the newly created fields to return in the response      {        id        gender        name {          given        }      }  }
    ```
  </Tab>

  <Tab title="Typescript">
    ```
    const patientAlias = await medplum.graphql(`  mutation {    # Define the fields for the resource being created, and alias as "newPatient"    newPatient: PatientCreate(      res: {        resourceType: "Patient"        gender: "male"        name: [            {                given: "Homer"            }          ]         }        )        # Specify which of the newly created fields to return in the response        {          id          gender          name {            given          }        }    }`);
    ```
  </Tab>
</Tabs>

<Accordion title="Example Response">
  ```
  {  data: {    newPatient: {      id: 'example-id',      name: {        given: 'Homer',      },      gender: 'male',    },  },};
  ```
</Accordion>

### Built-in types[​](#built-in-types "Direct link to Built-in types")

Medplum's graphQL schema provides type definitions for complex nested fields (aka "[Backbone Elements](https://www.hl7.org/fhir/R4/backboneelement.html#BackboneElement)"). These can be used to simplify arguments to your custom mutations.

This examples demonstrates how to create a mutation that creates a [`Communication`](/docs/api/fhir/resources/communication) resource and takes an array of `CommunicationPayload` types as a parameter:

<Tabs>
  <Tab title="GraphQL">
    ```
      # Use the built-in type `CommunicationPayloadCreate` as a parameter  mutation CreateCommunicationWithPayload($payload: [CommunicationPayloadCreate!]!) {  CommunicationCreate(res: {    resourceType: "Communication",    status: "draft",    payload: $payload  })  # Specify which of the newly created fields to return in the response  {    id,    resourceType,    payload {      contentString,      contentAttachment {        url      }    }  }}
    ```
  </Tab>

  <Tab title="Typescript">
    ```
    const communication = await medplum.graphql(`# Use the built-in type CommunicationPayloadCreate as a parametermutation CreateCommunicationWithPayload($payload: [CommunicationPayloadCreate!]!) {  CommunicationCreate(res: {    resourceType: "Communication",    status: "draft",    payload: $payload  })  # Specify which of the newly created fields to return in the response  {    id,    resourceType,    payload {      contentString,      contentAttachment {        url      }    }  }}`);
    ```
  </Tab>
</Tabs>

## Patch Mutation[​](#patch-mutation "Direct link to Patch Mutation")

You can partially update a resource using the `[resourceType]Patch` mutation. This mutation takes an `id` and a `patch` array of operations (see [RFC 6902](https://datatracker.ietf.org/doc/html/rfc6902)).

<Tabs>
  <Tab title="GraphQL">
    ```
    mutation {PatientPatch(  id: "example-id"  patch: [    { op: "replace", path: "/name/0/family", value: "Smith" },    { op: "replace", path: "/gender", value: "male" }  ]) {  id  gender  name { family given }}}
    ```
  </Tab>

  <Tab title="Typescript">
    ```
    const result = await medplum.graphql(`mutation {PatientPatch(  id: "example-id"  patch: [    { op: "replace", path: "/name/0/family", value: "Smith" },    { op: "replace", path: "/gender", value: "male" }  ]) {  id  gender  name { family given }}}`);
    ```
  </Tab>
</Tabs>

> **Note:** The `value` field in the patch operation is always a string due to GraphQL input type limitations. For complex values, encode as JSON strings and decode server-side if needed.

<Accordion title="Example Response">
  ```
  {"data": {  "PatientPatch": {    "id": "example-id",    "gender": "male",    "name": [      { "family": "Smith", "given": ["Alice"] }    ]  }}}
  ```
</Accordion>
